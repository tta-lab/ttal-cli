Coverage gap pattern (revised post ei-job-899): The "needs explicit trigger" bucket splits into two distinct groups. (1) APPLE-LOCKED — REFUND, DID_FAIL_TO_RENEW, OFFER_REDEEMED, all silent-drop / audit-only variants (RESCIND_CONSENT, METADATA_UPDATE, MIGRATE, RENEWAL_EXTENDED, EXTERNAL_PURCHASE_TOKEN, PRICE_INCREASE, CONSUMPTION_REQUEST), and the Apple duplicate-UUIDs bug. Apple's requestTestNotification only delivers TEST; there is no API to drive arbitrary types. app-store-server-library does not expose local-signing keys. Realistic ceiling for live verification = production-shadow JWS corpus once we ship and capture real traffic. (2) NOT-RUN but achievable — S7 cold-start, fresh-user-never-subscribed, V4–V8 (curl-craftable bodies), stub-level boundary tests, concurrency tests. These can land via stub-test, curl, or iOS flow without Apple cooperation.
